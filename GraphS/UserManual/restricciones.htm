Existe un caso especial en el que el usuario puede tener problemas ya que hay que operar de un modo que no es muy intuitivo.

 

Al generar un grafo mediante la operación union de dos grafos, el usuario debe especificar los dos grafos que desea unir, así como un nodo del primero grafo, un nodo del segundo grafo, y una arista para unir ambos nodos. Para evitar situaciones en las que la operación de union genere nodos y/o aristas repetidos, todas las variables serán renombradas por lo que el usuario deberá referirse a ellas por su nuevo nombre. Este nombre es: el nombre que contenía antes la variable concatenado con el identificador del grafo al que pertenece.

 

A continuación se muestra un ejemplo concreto, preste especial atención a los argumentos de la operación shortestPath que se define en el grafo con identificador grafo3:

 

node C, D;

edge Z;

 

graph grafo1 {

       node A, B, E, F;

       edge Y, X, H;

       Y = A -> B;

       Z = E -> F;

       X = B -> E;

       H = E -> A;

       op shortestPath(A,F);

       op minimumSpanningTree(A);

}

 

graph grafo2 {

       node A, B;

       edge Y, U, O, P;

       Y = A -> B;

       U = C -> A;

       O = D -> C;

       P = D -> B;

       op shortestPath(A,D);

}

 

graph grafo3 {

       union(grafo1,grafo2,A,B,Z);

       op shortestPath(Agrafo1,Bgrafo2);

}

 

Además, el usuario debe notar que la arista empleada para generar un grafo mediante la unión de otros dos grafos ha de ser declarada globalmente.

 

Si se produce algún error durante la generación del grafo, éste no será representado en el documento final.
